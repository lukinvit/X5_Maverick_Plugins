# MEMORY_CONVENTION.md — единая конвенция персистентной памяти агентов

> Адаптировано из X5_BM mothership@c87e9ee6 (2026-06-13). Ядро портировано, пути локализованы.

Status: ДЕЙСТВУЮЩАЯ (введена 2026-06-17).
Scope: все агенты `Product_agents/`. Индекс «агент → память → статус» — в `Product_agents/Dev_Agents/AGENTS.md` → «Team Status & Memory Index».
Профили агентов НЕ дублируют эти правила — только ссылаются сюда (допустима одна строка-указатель + агент-специфичные дельты). При расхождении конвенция главнее.

## 1. Имена файлов

- Канонический паттерн: `Product_agents/Dev_Agents/memmory_<agent>.md`.
- Написание **`memmory`** (двойное «m») — закреплённая историческая опечатка-конвент, унаследованная из mothership. НЕ «исправлять»: инструменты и памяти ссылаются на эти пути.
- Архив: каталог `memmory_<agent>_archive/` рядом с компактом; файлы `<YYYY-MM>.md`.

## 2. Двухуровневая структура

**Компакт** (`memmory_<agent>.md`) — единственный файл, который агент обязан читать перед прогоном. Hard-лимит: **1500 строк** (`wc -l`). Содержит:

- **durable-слой** (НЕ ротируется): правила памяти, снимок личности, recurring patterns / issue clusters, известные слепые зоны, standing-rule checklists, routing, промахи и коррекции, lessons learned, open backlog, self-audit checklist;
- **журнальный слой:** минимум **последние 3 прогона** (полные записи);
- ⤵-указатели на отротированные блоки (что и куда переехало, с датой ротации).

**Архив** (`memmory_<agent>_archive/*.md`) — полная история. Читается ТОЧЕЧНО (grep по `#issue` / дате), целиком НЕ читается и в read-before-run НЕ входит.

## 3. Инварианты (данные sacred — P-1)

1. **Append-only.** Ничего не удаляется и не редактируется задним числом. Опровергнутая запись помечается `superseded: <дата> → see <new entry>`.
2. **Ротация = перенос, не правка.** Строки переезжают в архив БЕЗ изменений (byte-exact), на их месте остаётся одна ⤵-строка-указатель. Сверка обязательна: `строк(оригинал) == строк(новый компакт) + строк(перенесено) − указатели`.
3. **Никаких секретов.** Токены XTracker, пароли, api-keys, session id, полные auth-заголовки — запрещены и в компакте, и в архиве, и в evidence (P-1).

## 4. Read-протокол (перед каждым прогоном)

1. Прочитать компакт **целиком** (он держится в лимите, чтобы это было физически выполнимо).
2. Без прочитанного компакта прогон не запускается.
3. Архив подключать точечно — по ссылкам из компакта или grep по конкретному вопросу.

## 5. Ротация — кто и когда

- **Сам агент**, в конце прогона: если компакт > 1500 строк — перенести старейшие журнальные записи (целые секции `## Run …`) в архивный файл месяца записи, оставить ⤵-указатель, выполнить сверку из §3.2, закоммитить вместе с memory-update.
- Durable-секции не ротируются никогда; если durable-слой распух — это сигнал на консолидацию (сгруппировать lessons по категориям), не на удаление.
